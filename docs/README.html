Threlmark

A local-first, central project & roadmap hub. Manage all your projects in one place and run a ranked, kanban-style roadmap for each — with the data stored as plain JSON on disk so your other tools can read a project's roadmap and write suggestions back into it.

Built with Next.js (App Router) + TypeScript. No database, no cloud, no accounts. Disk JSON under ~/.threlmark is the single source of truth; a small REST API sits over the same store so the UI and external tools share one path. A fully illustrated version of these docs lives in threlmark-docs.html.

Quick start

npm install
npm run dev        # http://localhost:3000  (or: PORT=4789 npm run dev)

npm run typecheck  # tsc --noEmit
npm run lint       # eslint
npm run build      # production build

All state is JSON on disk under the data root, which defaults to ~/.threlmark and can be overridden with THRELMARK_DATA_DIR. The root is home-based, not repo-relative — it's a shared hub every one of your apps points at.

Scoring & priority

priority = max(0, round(impact·3 + evidence·2 + fit·2 − effort·1.5))

Reused verbatim from the original Roadmap Lab, so imported cards rank identically (e.g. 5/5/5/4 → 29). Portfolio ranking weights priority by status — development 1.3, ranked 1.0, idea 0.85, done 0.15 — with a small boost for items that block others.

Flow & WIP

The board is a flow view, not just status tracking. Each item's append-only transitions[] history powers work-item age (time in lane; flagged stale past development 7d / ranked 21d / idea 60d), cycle time (Development → Done), and throughput (items finished per week). WIP limits in project.json flag overloaded lanes. A handoff is a first-class flow event: recording one (Handoff tab → Generate & mark handed off) stamps items with handoff and lets the Flow tab measure brief → shipped, agent throughput, and stalled briefs (handed off, not Done after 7 days). The home page shows a portfolio-wide flow strip.

Agent report-back closes the loop. The brief includes a reporting protocol so Claude/Codex report status by themselves — started / done / blocked / failed — via the REST API or a dropped reports/ file. A done report auto-moves the card to Done and the board shows a live toast; no manual step.

The shared data contract

~/.threlmark/
  threlmark.json                 # manifest
  links.json                     # cross-project dependency graph
  projects/<projectId>/
    project.json                 # project metadata
    board.json                   # lane ordering (self-healing)
    items/<itemId>.json          # card + transitions[] + handoff
    suggestions/<sugId>.json     # external drop zone = the Inbox
    suggestions/.dismissed/      # accepted/dismissed (audit)
    handoffs/<handoffId>.json    # recorded agent handoffs
    reports/<file>.json          # agent report-back (ingested on read)
    ROADMAP.md                   # regenerated human mirror
  shared/items/<itemId>.json     # shared across projects
  archive/projects/<projectId>/  # archived (still readable)

Per-item files let external tools write concurrently without clobbering a shared array; board.json self-heals on read. Atomic temp-then-rename writes; tolerant reads default missing fields and preserve unknown keys. Only source and title are required in a suggestion.

Features

  • Multi-project — create / archive / switch; cross-project Portfolio.
  • Ranked kanban — four lanes, drag, 4-axis scoring + computed priority, categories.
  • Roadmap Lab workflow — search, category filter, Rank by score, Push top 3, inline Add Item, on-card sliders, selection → live Queue/Markdown/JSON brief, Copy dev brief.
  • Import — read the original roadmap.html defaults (idempotent).
  • Handoff — file-scoped Claude/Codex prompts with acceptance checkboxes + verify commands, or Markdown / JSON.
  • Inbox + cross-project — accept/dismiss/promote suggestions; move, link, and share items across projects.
  • Flow — WIP limits, age/stale badges, a Flow tab (throughput, cycle time, aging) and a portfolio flow strip.
  • Agent flow — record handoffs to Claude/Codex; track brief → shipped, agent throughput, stalled briefs.
  • Local-first — no cloud, no accounts, no secrets.

Ecosystem: Threlmark + IdeaClyst

Threlmark's on-disk layout is a contract other local-first tools read and write. IdeaClyst (an idea engine) added a src/lib/threlmark/ layer that reads a project's roadmap + computes a gap map, and writes scored suggestions (Features / Spin-offs / Services, grounded in web research, tagged source:ideaclyst) into suggestions/ — via disk or the REST API, with disk fallback. They form a loop: IdeaClyst proposes what to build → it appears in Threlmark's Inbox → you accept & rank → hand off to an agent → the agent builds and reports back → Done. Any tool that speaks the contract can join.
